# PDCAサイクル（plan-do-check-act cycle）

品質管理など業務管理における継続的な改善方法。

**PDCAの各ステップ**

1. **Plan（計画）**: 過去の実績やデータに基づいて、具体的計画・目標を策定する。
2. **Do（実行）**: 計画に基づいて対策の導入・運用を行う。
3. **Check（評価）**: 実施した結果を評価・分析する。内部監査、外部監査などを行う。
4. **Action（改善）**: 評価に基づき業務の修正・改善を行い、次回に反映する。通常の組織では経営陣の権限と協力が必要になる。

最後のActを次のPDCAサイクルにつなげ、螺旋を描くように1周ごとに各段階のレベルを向上（スパイラルアップ、spiral up）させて、継続的に業務を改善する。

**成功のポイント**

- **Check（評価）を重視する**: 計画と実行だけで終わらず、必ず振り返りを行う。
- **具体的な数値目標を設定する**: 定量的な指標（KPI）を用いることで、成果が可視化される。 


> PDCAサイクルの考え方は、品質マネジメントシステム [ISO 9001](https://ja.wikipedia.org/wiki/ISO_9000 "ISO 9000")、環境マネジメントシステム [ISO 14001](https://ja.wikipedia.org/wiki/ISO_14000#ISO_14001 "ISO 14000")、情報セキュリティマネジメントシステム [ISO 27001](https://ja.wikipedia.org/wiki/ISO/IEC_27002#ISO/IEC_27000_%E3%82%B7%E3%83%AA%E3%83%BC%E3%82%BA "ISO/IEC 27002")、個人情報保護マネジメントシステム [JIS Q 15001](https://ja.wikipedia.org/wiki/JIS_Q_15001 "JIS Q 15001")などに用いられている。
