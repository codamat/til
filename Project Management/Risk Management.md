# リスクマネジメント（Risk Management）

リスクとチャンスをコントロールする役割

PMBOKにおけるリスクマネジメントは、プロジェクト目標（納期、コスト、品質等）に影響を与える不確実な事象（脅威または機会）を特定、分析し、体系的に対応・監視するプロセス。

## 主要プロセス
1. 立ち上げ： 
2. **計画：** 
   - **リスクの特定 (Identification):**  
    事業に悪影響を及ぼす可能性のあるリスク（自然災害、サイバー攻撃、法的リスク、事故など）を洗い出す。
   - **リスクの分析・評価 (Analysis/Evaluation):**  
    特定したリスクを「発生頻度」と「発生時の影響度」の2軸で評価し、優先順位を決める。
3. **実行：** 
   - **リスク対応 (Treatment):** 
     リスクに対して、「回避（やめる）」「低減（対策する）」「移転（保険をかける）」「受容（許容する）」のいずれかの対応をとる
4. **コントロール:** 
   - **リスクの監視:** リスクの発生状況を追跡し、対応策の有効性を確認する
5. 終結：


## リスクマネジメントの7つのプロセス


1. **リスクマネジメントの計画**: 管理の手法や役割、予算などを決める。
2. **リスクの特定**: どのようなリスクがあるかを洗い出す。
3. **定性的リスク分析**: 発生確率や影響度を主観的（大・中・小など）に評価し、優先順位をつける。
4. **定量的リスク分析**: 数値データを用いて、プロジェクト全体への影響を客観的に評価する。
5. **リスク対応の計画**: リスクが起きた時の戦略を立てる。
6. **リスク対応策の実行**: 計画した対応策を実際に実施する。
7. **リスクの監視**: 新たなリスクの発生や対応策の効果をチェックし続ける。 

## マイナスリスク（ネガティブリスク）に対して

- **回避**: 計画を変更し、リスクの原因そのものを排除する。
- **転嫁（移転）**: **保険への加入や外注**などにより、第三者にリスクの責任を負わせる。
- **軽減**: 発生確率や影響度を下げるための対策を講じる。
- **受容**: 特別な対策はせず、発生した際に予備費などで対応する

## チャンス（ポジティブリスク）に対して

| 戦略               | 内容                          | 具体例                               |
| ---------------- | --------------------------- | --------------------------------- |
| **活用 (Exploit)** | 確実に発生するように働きかける（確率を100%にする） | 熟練度の高いメンバーを投入し、工期短縮を確実に実現させる      |
| **共有 (Share)**   | メリットを享受できる第三者に一部を割り当てる      | 自社だけでは活かせない好機を、パートナー企業と共同で進める     |
| **強化 (Enhance)** | 発生確率や影響度を高める                | チームにインセンティブを出し、作業効率が上がる可能性をさらに高める |
| **受容 (Accept)**  | 積極的な対策はせず、発生した際にその恩恵を受ける    | 偶然コストが下がったら、その分を予備費に回す程度に留める      |
| **エスカレーション**     | プロジェクトの範囲外のため、上位組織に委ねる      | 経営層でないと判断できないような全社的なビジネスチャンスを報告する |

### なぜポジティブリスクを管理するのか？

単に「ラッキー」で済ませず、計画的に「活用」や「強化」を行うことで、プロジェクトの**価値を最大化**できる。  
不確実性を単なる「脅威」として守るだけでなく、「好機」として攻めに転じる姿勢がPMBOKのリスクマネジメントの本質

## チャンスとリスクの対比

| チャンス             | 対比  | リスク                      |
| ---------------- | --- | ------------------------ |
| **活用 (Exploit)** | ←→  | **回避**（**Avoid**）        |
| **共有 (Share)**   | ←→  | **転嫁（移転）**（**Transfer**） |
| **強化 (Enhance)** | ←→  | **軽減**（**Mitigate**）     |
| **受容 (Accept)**  |  =  | **受容**（**Accept**）       |
